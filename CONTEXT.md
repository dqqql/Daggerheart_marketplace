# Daggerheart Marketplace Directory

这是一个用于展示 Daggerheart 第三方模组与资源的目录站。它不承载资源正文，也不处理交易，只负责聚合展示、筛选与跳转。

## Language

**目录站**:
只展示资源摘要并跳转到外部内容地址的网站。
_Avoid_: 商城, 内容站, 下载站

**资源条目**:
首页中的一个展示卡片，包含封面、标题、作者、标签、推荐值、跳转链接与可选简介。
_Avoid_: 商品, 帖子, 文章

**封面资源**:
保存在目录站服务器本地、用于展示资源条目封面的图片文件。
_Avoid_: 图床链接, 外链缩略图

**内容标签**:
用于描述资源条目承载什么游戏内容的一组标签，如模组、敌人、职业。
_Avoid_: 主分类, 栏目

**风味标签**:
用于描述资源条目题材或氛围风格的一组标签，如西幻、科幻、武侠。
_Avoid_: 主分类, 世界观正文

**推荐值**:
由管理员手动维护的数值字段，当前以 0/1 表示是否推荐，后续可扩展为更细的推荐强度。
_Avoid_: 普通标签, 布尔开关

**自由标签**:
由管理员在录入资源条目时直接输入、而非从预设词库中勾选的标签。
_Avoid_: 固定词库, 受控枚举

**标签规范化**:
对管理员输入的标签做最小清洗，包括去首尾空格、统一空白形式与同条目内去重。
_Avoid_: 同义词治理, 词库强控

**简介**:
资源条目的简短说明文字，用于鼠标悬停时展示，可为空。
_Avoid_: 长正文, 详情页内容

**条目投稿**:
社区成员在首页通过公开表单提交资源条目的行为，进入待审核数据，不直接公开。
_Avoid_: 公开发布, 自动收录

**反馈邮箱**:
投稿者可选填写的联系邮箱，仅用于管理员驳回投稿时发送审阅意见，不进入公开资源条目。
_Avoid_: 用户账号, 公开联系方式

**审阅意见**:
管理员驳回投稿时写给投稿者的修改说明，可通过邮件发送到反馈邮箱。
_Avoid_: 评论, 公开评价

**驳回邮件**:
管理服务通过外部邮件服务发送给反馈邮箱的审核结果通知。
_Avoid_: 自建邮箱服务器, 营销邮件

**全文搜索**:
首页搜索框对资源条目的标题、作者、标签与简介进行匹配。
_Avoid_: 只搜标题, 复杂检索语法

**冻结工具栏**:
固定在首页顶部的筛选栏，用于承载搜索框、内容标签筛选与风味标签筛选。
_Avoid_: 滚动后消失的筛选区, 分散筛选入口

**标签输入框**:
管理入口中用于录入标签的交互控件，管理员输入词语后生成可删除的标签 chip。
_Avoid_: 单个大文本框, 纯逗号分隔长字符串

**标签候选集**:
首页筛选可供点击的标签列表，由现有资源条目的标签自动汇总并去重生成。
_Avoid_: 独立词库, 手工维护标签表

**标签计数**:
某个标签当前关联的资源条目数量，在筛选界面中以如"西幻(5)"的形式展示。
_Avoid_: 点赞数, 浏览量

**玩家点赞**:
访客对资源条目的单向认可行为，纯点赞无踩，通过 IP 去重，无需账号登录。
_Avoid_: 评分, 投票, 收藏

**点赞数**:
某个资源条目累计收到的玩家点赞数量，保存在条目数据中，用于影响首页排序。
_Avoid_: 评分值, 热度值

**标签弹窗**:
点击工具栏中的标签筛选框后展开的完整标签面板，用于显示全部可筛选标签。
_Avoid_: 新页面, 下拉长列表

**条目数据文件**:
保存在服务器本地、以 JSON 格式存储资源条目配置的数据文件。
_Avoid_: 数据库表, 用户行为日志

**管理服务**:
一个极小的 Python 服务，仅负责口令校验、封面上传与条目数据写入。
_Avoid_: 完整后端系统, 业务平台

**管理入口**:
用于新增、编辑、排序或下架资源条目的轻量配置界面。
_Avoid_: 后台系统, 用户中心

**管理路径**:
独立于首页的管理入口访问地址，用于承载口令校验与条目维护界面。
_Avoid_: 首页弹窗后台, 资源详情页

**访问口令**:
进入管理入口时输入的一段共享密码。
_Avoid_: 账号密码, 用户登录

**管理会话**:
管理员输入访问口令后，由服务端签发的一段短期登录状态，用于后续管理操作。
_Avoid_: 永久登录, 每次操作重输口令

**条目删除**:
管理员将资源条目从目录站数据中移除，而不是改成隐藏状态。
_Avoid_: 下架, 软删除

## Relationships

- 一个**目录站**展示多个**资源条目**
- 每个**资源条目**对应一个外部跳转链接
- 每个**资源条目**可以关联一个**封面资源**
- 每个**资源条目**可以拥有多个**内容标签**
- 每个**资源条目**可以拥有多个**风味标签**
- 每个**资源条目**可以带有一个**推荐值**
- 每个**资源条目**可以有可选的**简介**
- **条目投稿**进入待审核数据，审核通过后才生成**资源条目**
- **条目投稿**可以有可选的**反馈邮箱**
- **反馈邮箱**不属于公开**资源条目**
- 管理员驳回**条目投稿**时可以填写**审阅意见**
- 有**反馈邮箱**且邮件服务已配置时，**管理服务**会发送**驳回邮件**
- **内容标签**与**风味标签**允许使用**自由标签**
- **自由标签**在保存前会经过最小的**标签规范化**
- **内容标签**可以为空
- **风味标签**可以为空
- 每个**资源条目**至少需要标题与外部跳转链接
- 首页的**全文搜索**会覆盖标题、作者、标签与**简介**
- 首页使用顶部的**冻结工具栏**承载搜索与两套标签筛选
- 首页的两套**标签候选集**均由现有资源条目自动汇总生成
- **冻结工具栏**中的标签筛选区域长度固定，只展示按**标签计数**排序后的头部标签
- 点击工具栏中的标签筛选区域后，会打开展示全部标签的**标签弹窗**
- 首页标签在筛选界面中会显示对应的**标签计数**
- **管理入口**读写本地的**条目数据文件**
- **管理入口**通过**管理服务**完成口令校验、封面上传与保存
- **管理入口**中的**内容标签**与**风味标签**分别通过独立的**标签输入框**录入
- **管理入口**负责维护**资源条目**
- **管理入口**通过单独的**管理路径**访问
- **访问口令**用于限制谁能进入**管理入口**
- **管理会话**用于在短时间内维持管理入口的登录状态
- **资源条目**当前只支持**条目删除**，不引入隐藏状态

## Example dialogue

> **开发者:** “这个**资源条目**点进去后，是站内详情页还是外链？”
> **领域负责人:** “没有二级页，**目录站**只负责展示，点击后直接跳到外部内容地址。”

## Flagged ambiguities

- “后台”曾被用来指代完整账号系统，现已明确为带**访问口令**的**管理入口**
- “前端密码”曾被用来指代纯静态校验，现已明确为前端输入、服务端校验的轻量方案
- “封面”曾可能指外部图片链接，现已明确为服务器本地保存的**封面资源**
- “评分/点赞机制”曾可能指用户互动，现已明确为纯点赞（无踩、无评论），使用 IP 记录去重，不需账号
- “管理功能”曾可能被理解为首页内嵌交互，现已明确为通过独立**管理路径**访问
- “数据存储”曾可能指数据库，现已明确为本地 JSON 形式的**条目数据文件**
- “分类/标签”曾可能混用，现已明确为两套并行体系：**内容标签**与**风味标签**
- “店长推荐”曾可能被理解为普通标签或布尔值，现已明确为独立数值字段 **推荐值**
- “评分”和“推荐”曾可能并存，现已明确合并，只保留数值字段 **推荐值**
- “标签管理”曾可能被理解为固定词库，现已明确为管理员可直接输入的**自由标签**
- “自由输入”曾可能意味着完全不处理，现已明确为保留自由输入，但执行最小的**标签规范化**
- “标签是否必填”现已明确：**内容标签**与**风味标签**都可空；当前只有标题与外部跳转链接必填
- “技术栈”曾可能被理解为纯静态站，现已明确为静态首页配合极小的 Python **管理服务**
- “一句话介绍”曾未出现在配置字段中，现已明确为 JSON 内可选的**简介**
- “投稿联系方式”现已明确为可选**反馈邮箱**，只存在于待审核数据，不公开展示
- “驳回原因”现已明确为管理员填写的**审阅意见**，可通过外部邮件服务作为**驳回邮件**发送
- “搜索范围”现已明确：标题、作者、标签与**简介**都会进入**全文搜索**
- “admin 登录”现已明确为输入一次口令后建立短期**管理会话**
- “条目状态”现已明确：不做 hidden，只支持**条目删除**
- “首页筛选 UI”现已明确为顶部**冻结工具栏**，内含搜索框与两套标签筛选
- “标签录入 UI”现已明确为两套独立的**标签输入框**，输入后生成可删除的 chip
- “筛选标签来源”现已明确为从现有资源自动汇总出的**标签候选集**
- “工具栏标签展示”现已明确为固定长度、按**标签计数**排序，只显示头部标签，点击后通过**标签弹窗**查看全集
- “首页排序/分区”尚未定案，当前先聚焦**标签系统**，展示编排留待看效果后再定
