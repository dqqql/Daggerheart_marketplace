# Web Vitals 用户体验改进计划

## 背景

Cloudflare Web Analytics 在 2026-07-05 至 2026-07-12 的真实用户数据中显示：

- LCP：54% 良好，P75 约 4.56 秒；主要 LCP 元素为目录卡片封面。
- INP：88% 良好；搜索框和卡片交互存在少量 300-500ms 样本。
- CLS：38% 良好，35% 较差；主要位移元素为页面主体和页脚。

样本量约 300-400，当前以修复明确的结构性问题为主，不针对极少数异常样本过拟合。

## 总原则

用户体验优先：优先减少用户能直接感受到的等待、跳动和交互迟滞，同时保留现有黑暗奇幻视觉风格和目录浏览方式。

## 实施顺序

### P0：稳定首屏

- 首次 HTML 直接提供与真实卡片同尺寸的骨架屏。
- 加载状态使用 `aria-live` 告知辅助技术，不额外占用可见布局。
- 数据加载期间为目录保留稳定高度，避免页脚进入首屏后又被推走。
- 为动态结果数量提供礼貌播报。

验收目标：正常网络和慢速 API 下，工具栏以下不出现“空白 -> 整页内容”的明显跳动。

### P0：加速首屏封面

- 按视口为首屏封面分配 eager/high priority：手机 1 张、平板 2 张、桌面 4 张。
- 其余封面继续延迟加载，所有封面异步解码。
- 输出明确的 3:4 图片尺寸，避免浏览器等待资源后才能确认比例。
- 无封面条目不再发出空图片请求。

验收目标：首屏只有视口内最可能成为 LCP 的封面获得高加载优先级，其余封面保持 lazy。

### P1：降低长目录和交互成本

- 卡片使用 `content-visibility` 跳过屏幕外渲染，同时提供内在尺寸占位。
- 卡片和筛选标签使用事件委托，避免每次筛选后为大量节点重复绑定监听器。
- 保留已有 200ms 搜索防抖。
- 点赞状态加载完成后原位更新按钮，不再重建整个目录。

验收目标：搜索、清除搜索、打开条目和点赞行为不变；点赞状态同步不会触发目录整体动画和重排。

### P1：可访问性与动效

- 为键盘操作提供清晰的 `focus-visible` 样式。
- 遵循 `prefers-reduced-motion`，关闭非必要的入场、扫光和过渡动画。

## 已有基础设施

- R2 封面已经返回一年 `immutable` 缓存，本轮无需重复修改。
- `/api/public/bootstrap` 已配置边缘缓存和 stale-while-revalidate。

## 上线后观察

- 至少收集一周数据，并优先查看首页 `/`，不要让 `/admin/` 样本干扰访客体验判断。
- 目标：CLS P75 小于 0.1，LCP P75 小于 2.5 秒，INP P75 小于 200ms。
- 若 LCP 仍由个别封面主导，下一阶段在上传链路生成 400-600px 的目录缩略图及响应式 `srcset`。
